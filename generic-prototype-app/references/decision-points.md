# Decision Points

## Ask First
1. What should the GitHub repo be called?
2. Should the repo be under the user’s personal account or an organization?
3. Do they want GitHub Pages publishing?
4. Do they need anything beyond a frontend prototype, such as backend, auth, database, or routing?
5. Do they want a plain starter or a UI system/component library added up front?

## Default Assumptions
- Stack: Vite + React + TypeScript + Tailwind
- Package manager: npm
- Visibility: private GitHub repository
- Branch: `main`
- Deployment: off unless explicitly requested
- Scope: frontend-only unless the user asks for more

## Important Warnings
- GitHub Pages content is public
- The skill should not start if the current folder already contains unrelated work
- The skill should not add backend or auth infrastructure by default
