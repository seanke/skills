# Template Discovery

Use this when the user is still deciding what template to use.

## Typical Questions

- What templates are available for this idea?
- Which template best matches this project shape?
- What parameters does this template support?
- What files will the template create?

## Workflow

1. Search or list templates.
2. Inspect template help for parameters and defaults.
3. Dry-run the best match before creating anything.
4. Compare the options if multiple templates are plausible.

## Good Signals

- `dotnet new search`
- `dotnet new list`
- `dotnet new <template> --help`
- `dotnet new <template> --dry-run`

Use this reference when choosing between template options, not when the project is already selected.
