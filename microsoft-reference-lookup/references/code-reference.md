# Microsoft Code Reference

Use this when the user needs API signatures, working samples, or SDK-specific code guidance.

## Good Fits

- Verify that a method or type exists
- Find a working code sample
- Compare an implementation against official usage
- Check the right package or namespace

## Workflow

1. Search for the API or scenario.
2. Fetch full details if there are overloads or complex parameters.
3. Check the sample against the current SDK version.
4. If the answer depends on both behavior and code shape, also read `references/docs.md`.

## Fallback

If the code-sample search tool is unavailable, use the Microsoft Learn CLI or official docs as the next best source.
