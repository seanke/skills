# Microsoft Docs

Use this when the user needs to understand a Microsoft technology rather than write code.

## Good Fits

- Concepts and architecture
- Tutorials and quickstarts
- Configuration options
- Limits and quotas
- Best practices

## Workflow

1. Search official documentation first.
2. Fetch the full page if the excerpt is not enough.
3. Prefer the exact product and version in the query.
4. Summarize the result with the doc source in mind.

## Fallback

If the docs tool is unavailable, use the Microsoft Learn CLI as a backup.

This reference is for explanation and policy, not for API signatures or sample code.
